**[NOVEL FRAMEWORK]:**  
We model the deterministic execution of a Turing machine $M$ on a 3SAT instance $\phi$ as a **traversal through a Proof Graph** $\mathcal{G}_\phi$ associated with $\phi$. The vertices of $\mathcal{G}_\phi$ are partial assignments (or configurations of a proof system such as Resolution or Polynomial Calculus). Directed edges correspond to valid inference steps that extend or modify the assignment. A **proof** of satisfiability is a directed path from the empty assignment $\emptyset$ to a satisfying total assignment $\sigma^*$. The 2D Cook–Levin Tableau $T_M(\phi)$ records the sequence of memory states of $M$ as it constructs such a path implicitly. At any time $t$, the active memory of $M$ (the non‑blank portion of the tape, bounded by $S = \text{poly}(n)$ cells) holds a **bounded‑width certificate**—a set of partial assignments whose combined size is at most $S$ bits.

The **algebraic degree** or **resolution width** of a partial assignment is the number of variables assigned. A classic result in proof complexity states that for random 3SAT instances near the threshold, any resolution refutation (or proof of satisfiability) requires **width** $\Omega(n)$ and **size** $2^{\Omega(n)}$. In our context, the TM must essentially simulate a proof of width $\Omega(n)$ using only $S = \text{poly}(n)$ memory. This is precisely the setting of a **pebble game** on the proof DAG: the TM can only store a bounded number of intermediate nodes (pebbles) and must repeatedly recompute forgotten nodes.

**[PROPOSED STEP 6]:**  
We prove the **Time–Space Proof Bottleneck**. Let $W = \Omega(n)$ be the minimum width of any proof of satisfiability for the expander instance $\phi$ in a given proof system (e.g., Resolution). The TM $M$ has a tape of $S$ cells, each holding a symbol from a finite alphabet. The total information that can be stored at any instant is $O(S \log|\Sigma|) = \text{poly}(n)$ bits. However, to verify a satisfying assignment, $M$ must at some point represent a partial assignment of width $\Omega(n)$. Since $S \ll 2^{\Omega(n)}$, $M$ cannot store all necessary information simultaneously. Instead, it must **play a pebble game** on the proof graph: it can only keep a small number of vertices in memory, and to advance it must drop some and later recompute them.  

In the pebble game, each time a node is dropped and later needed again, it must be recomputed from scratch, leading to an exponential blow‑up in time. Formally, for a proof graph of depth $D$ and width $W$, any pebbling strategy with $S$ pebbles requires at least $2^{\Omega(W/S)}$ moves. Here $W = \Omega(n)$ and $S = \text{poly}(n)$, so $W/S = \Omega(n/\text{poly}(n))$, yielding a lower bound of $2^{\Omega(n)}$ steps. Each move corresponds to a time step of $M$ (or at most a polynomial overhead). Therefore, $T \ge 2^{\Omega(n)}$.

Thus any deterministic polynomial‑time Turing machine fails to solve all such satisfiable expander instances, implying $\mathbf{P} \neq \mathbf{NP}$.

**[BARRIER EVASION CHECK]:**  
- **Relativization (Baker–Gill–Solovay):** An oracle Turing machine $M^O$ can query an oracle for SAT. The oracle provides the satisfying assignment $\sigma^*$ directly, without requiring $M^O$ to traverse the proof graph. In terms of the pebble game, the oracle acts as a **magic pebble** that instantly supplies the target node, bypassing the width and depth constraints. The Time–Space Proof Bottleneck bound assumes that $M$ must construct the proof internally; an oracle destroys this assumption. Since our lower bound relies on the unrelativized machine's need to navigate the proof graph with bounded memory, and this need vanishes with an oracle, the proof is **non‑relativizing**.  
- **Natural Proofs:** The hard instances are random 3SAT formulas at the critical threshold, whose proof complexity lower bounds are well‑known. The property of having high resolution width is a syntactic property of the instance, not of the Boolean function computed (which is constant‑1 for satisfiable instances). The set of such instances is sparse and not polynomial‑time recognizable. Therefore, the argument evades the largeness and constructivity criteria of Razborov–Rudich.
